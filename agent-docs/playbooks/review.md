# Playbook - Code Review

## Focus

Priorita a bug, regressioni, rischi operativi e gap di test.

## Flusso

1. Leggi contesto (issue/PR/descrizione).
2. Analizza diff per severita.
3. Riporta findings con file/linea e proposta di fix.
4. Evidenzia rischi residui e test mancanti.

## Output atteso

- Findings ordinati per severita.
- Nessun finding: dichiararlo esplicitamente.
- Sommario secondario breve.
